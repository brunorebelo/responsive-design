Reflowing layouts

  • Mostly fluid
    • Most popular pattern
    • Layout pattern doesnt change until small screen
    • Fluid grid and images scale down until smallest screen size (becomes stacked)
  • Column drop
    • Also popular, keeps main contents grouped by importance
  • Layout shifter
    • Not as popular because it does a lot of work (different layouts for small, mediumn and large screens)
    • e.g. Boston Globe

Navigation

  • Bottom-anchored footer
    • No JS depedency
    • Easy to implement
    • Scales up easily without much CSS
    • Anchor/page jump
  • Top stacked links
    • Takes up a lot of space
    • Easy to implement, no JS required
  • Select navigation
    • Frees up more space than horizontal/vertical navs
    • Can use native controls/responses to up/down actions
    • JS depedency
    • Hard to style
    • Confusing? Select menus usually used in form context
  • Toggle navigation
    • Easy to scale up
    • No page jumps
    • JS dependency (minimal)
    • CSS animations (rendering issues on Android)
    • *needs to be tested on older phones
